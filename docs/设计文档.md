# AMS 配置管理系统 — 设计文档

---

## 一、项目背景

### 1.1 原有系统

本项目替代原 **HldAssistant.exe**（Qt5 C++ + QML 桌面程序）的配置管理功能。原程序主要功能：

- 通过 **NMEA 协议**采集船舶航行数据（GPS、AIS、GYRO、EchoSounder 等）
- 通过 **Modbus TCP/RTU** 采集机舱 AMS 报警监控数据
- 将数据存入 MySQL 数据库
- 通过 Kafka 转发给岸端平台

### 1.2 为什么要替换

新系统聚焦原程序的**配置管理**模块，解决以下痛点：

| 痛点 | 新方案 |
|---|---|
| 每条船的 AMS 厂商不同，Excel 格式各不相同 | 灵活的列映射导入 + 可保存的导入模板 |
| 厂商频繁修改工况点，人工比对 Excel 找差异 | 自动版本比对 + 变更审批 |
| 只能 Windows 桌面用 | Web 版，任何设备浏览器访问 |
| 修改配置要登录服务器改文件 | 网页上直接操作 |

## 三、数据模型关系图

```
ships（船舶）
  │
  ├── gateways（网关：SailingData / EngineData）
  │     │
  │     └── channels（通道）
  │           │
  │           └── modbus_configs（Modbus客户端配置）
  │                 │
  │                 ├── signal_points（工况点/信号点）
  │                 │     └── config_versions（配置版本）
  │                 │           └── version_changes（变更明细）
  │                 │
  │                 └── import_templates（Excel导入模板）
```

### 3.1 核心表说明

| 表名 | 说明 | 对应原系统 |
|---|---|---|
| `ships` | 船舶信息 | 新增（原系统无船舶管理） |
| `gateways` | 数据通路（网关） | `ways` 表 |
| `channels` | 数据通道 | `channelsdataprotocol` |
| `modbus_configs` | Modbus 设备配置 | 原 JSON 配置中的 `NetModTcp` |
| `signal_points` | 每个工况点 | 原 Excel 中的每一行 |
| `config_versions` | 配置版本记录 | 新增 |
| `version_changes` | 版本间变更明细 | 新增 |
| `import_templates` | Excel 列映射模板 | 新增 |

### 3.2 工况点（signal_points）字段详解

| 字段 | 说明 | 示例 |
|---|---|---|
| `name` | 工况点名称 | "监测报警1#AC220V输入故障" |
| `modbus_address` | Modbus 寄存器地址 | "0" 或 "0-1" |
| `signal_type` | 信号类型：开关量/模拟量/浮点数 | "开关量" |
| `unit` | 单位（模拟量专用） | "℃"、"Bar"、"Kpa" |
| `data_type` | 数据类型 | "浮点数"、"整数" |
| `byte_order` | 字节顺序 | "1234"、"4321" |
| `coefficient` | 系数公式 | "0,1,0" |
| `ams_module_id` | AMS 模块编号 | "U0_DI00_02" |
| `is_active` | 是否有效（1有效/0备用） | 1 |

## 四、页面功能详细说明

### 4.1 数据库操作页面 `/database`

**流程**：输入 MySQL 连接信息 → 测试连接 → 底部状态条反馈

**表单字段**：
- 数据库 IP 地址（`host`）
- 端口号（`port`，默认 3306）
- 用户名（`username`）
- 密码（`password`）
- 数据库名（`database`）

**交互**：
- 点击"连接数据库"按钮 → POST `/api/database/test-connection`
- 成功：状态条变绿 🟢"数据库已连接"
- 失败：状态条变红 🔴"数据库未连接" + 错误提示

### 4.2 机舱设备页面 `/engine-room`（核心页面）

**布局**：

```
┌─────────────────────────────────────────────────┐
│  船舶下拉框  │  网关下拉框  │  Modbus客户端下拉框  │ +新增按钮 │
├──────────────────────┬──────────────────────────┤
│  📡 基础通讯配置       │  📋 工况点列表           │
│                      │                          │
│  通讯波特率 │ 数据位   │  序号 │ 名称 │ 地址 │ ... │
│  校验位   │ 停止位   │                          │
│  从站ID   │ 功能码   │                          │
│  通讯类型 │ 电气接口  │                          │
│  IP地址   │ 端口    │                          │
│                      │                          │
│  📋 寄存器配置         │                          │
│                      │                          │
│  起始地址 │ 寄存器数量 │                          │
│  数据类型 │ 数据长度   │                          │
├──────────────────────┴──────────────────────────┤
│                              🔴 生成配置 （右下角）  │
└─────────────────────────────────────────────────┘
```

**操作流程**：

1. 选择船舶 → 自动加载该船的网关列表
2. 选择网关 → 自动加载该网关的 Modbus 客户端
3. 选择/新增 Modbus 客户端
4. 配置基础通讯参数 → 点击"保存"
5. 配置寄存器参数
6. 管理工况点（手动添加 / Excel 导入）
7. 点击红色"生成配置"按钮 → 生成 JSON + TXT 文件

### 4.3 Excel 导入流程（核心功能）

```
用户操作                         系统处理
─────────                     ──────────
上传 AMS 厂商 Excel  ──────→  解析并预览前20行
                                    │
选择/创建列映射模板  ←──────   显示表头列名
（拖拽匹配：名称=B列                  │
  Modbus地址=C列 ...）               │
                                    │
点击"导入并比对"  ─────────→  根据模板解析全部数据
                                    │
                           创建新版本 ConfigVersion
                                    │
                           与上一个版本比对
                                    │
显示版本差异报告  ←───────  🟢 新增 N 条
  （逐条审批）                  🔴 删除 N 条
                                🟡 修改 N 条
                                    │
点击"全部接受"  ─────────→  写入变更记录到数据库
                                    │
                           更新 signal_points 表
                ✅ 完成
```

### 4.4 配置入库页面 `/cfg2db`

**流程**：
1. 输入目标数据库连接信息（IP/端口/用户名/密码/数据库名）
2. 选择待入库的 Modbus 客户端配置
3. 点击"配置入库"按钮
4. 系统将配置数据写入目标数据库的 `channelsdataprotocol` 表

**写入格式**：与原有 HldAssistant 生成的 JSON 格式完全兼容

### 4.5 导入模板管理页面 `/templates`

管理所有已保存的 Excel 列映射模板，支持：
- 查看模板列表（显示模板名称、AMS 厂商、船型系列、列映射）
- 新建模板
- 编辑模板
- 删除模板

**模板字段**：
| 字段 | 说明 |
|---|---|
| `name`（必填） | 模板名称，如"远东德华 AMS" |
| `manufacturer` | AMS 厂商名称 |
| `ship_series` | 船型系列 |
| `column_mapping` | 列映射 JSON：系统字段 → Excel 列字母 |
| `header_row` | 表头所在行号（默认 7） |
| `data_start_row` | 数据起始行号（默认 8） |

## 六、版本比对逻辑

### 6.1 比对规则

每次导入 Excel 时，创建一个新版本（`ConfigVersion`），与**前一个版本**进行比对。

**匹配依据**：`ams_module_id`（AMS 模块编号）或 `modbus_address`（寄存器地址）

| 情况 | 判定 |
|---|---|
| 新版有，旧版无 | 🟢 新增 |
| 旧版有，新版无 | 🔴 删除 |
| 都有，但名称/地址/类型等变了 | 🟡 修改 |

### 6.2 比对字段

- `name` — 工况点名称
- `modbus_address` — Modbus 地址
- `signal_type` — 信号类型
- `unit` — 单位
- `data_type` — 数据类型

### 6.3 审批流程

```
变更记录 status = "pending"
        │
   ┌────┴────┐
   │         │
 全部接受   全部拒绝
   │         │
 status →   status →
"accepted"  "rejected"
```
