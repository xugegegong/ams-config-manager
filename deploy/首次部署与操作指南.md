# 首次部署与操作指南

---

## 一、确认 Runner 是否在线（GitHub 页面）

### 方式 1：通过仓库设置查看

1. 打开你的仓库：**https://github.com/xugegegong/ams-config-manager**
2. 点击顶部导航栏的 **Settings**（设置）
3. 左侧菜单找到 **Actions** → **Runners**（运行器）
4. 如果列表中有你的服务器，且状态显示 **🟢 Idle**，就说明 Runner 正常在线

### 方式 2：通过 Actions 日志查看

提交代码后，在仓库页面点击 **Actions** 标签，可以看到运行记录。

## 二、首次手动部署（服务器上执行）

由于仓库还没有项目代码，需要手动克隆并部署一次，后续就可以全自动了。

### 第 1 步：克隆代码

```bash
cd /tmp
git clone https://github.com/xugegegong/ams-config-manager.git
cd ams-config-manager
```

### 第 2 步：安装后端依赖

```bash
pip3 install -r backend/requirements-prod.txt
```

### 第 3 步：构建前端

```bash
npm --prefix frontend ci --registry=https://registry.npmmirror.com
npm --prefix frontend run build
```

### 第 4 步：同步文件到部署目录

```bash
sudo mkdir -p /opt/ams

sudo rsync -av --delete \
  --exclude='__pycache__' \
  --exclude='*.pyc' \
  backend/ /opt/ams/backend/

sudo rsync -av --delete \
  frontend/dist/ /opt/ams/frontend/dist/
```

### 第 5 步：复制系统服务配置

```bash
sudo cp deploy/ams-backend.service /etc/systemd/system/
sudo cp deploy/nginx-ams.conf /etc/nginx/sites-available/ams

# 启用 nginx 配置
sudo ln -s /etc/nginx/sites-available/ams /etc/nginx/sites-enabled/ 2>/dev/null || true
```

### 第 6 步：修改数据库配置

```bash
sudo vi /opt/ams/backend/app/core/config.py
```

将 `DATABASE_URL` 改为 MySQL 地址，例如：

```python
DATABASE_URL = "sqlite:///./ams_config.db"   # 先用 SQLite 测试
# DATABASE_URL = "mysql+pymysql://用户名:密码@localhost:3306/ams_config"  # 正式用 MySQL
```

### 第 7 步：启动后端服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable ams-backend
sudo systemctl start ams-backend
sudo systemctl status ams-backend
```

### 第 8 步：配置 Nginx 并启动

```bash
sudo nginx -t                    # 检查配置是否正确
sudo systemctl reload nginx      # 重载 nginx
```

### 第 9 步：验证部署是否成功

```bash
# 测试后端 API
curl http://localhost:8000/api/health

# 期望输出：{"status":"ok","version":"1.0.0"}
```

### 第 10 步：初始化管理员账号

```bash
curl -X POST http://localhost:8000/api/auth/init-admin
# 期望输出：{"message":"Admin created (username=admin, password=admin123)"}
```

## 三、访问系统

| 服务 | 地址 |
|---|---|
| 前端页面 | `http://服务器IP`（如果配了 nginx） |
| 后端 API | `http://服务器IP:8000` |
| API 文档 | `http://服务器IP:8000/docs` |

**默认登录账号**：`admin` / `admin123`

## 四、后续自动部署

首次部署完成后，以后每次更新代码只需要：

```bash
# 本地修改代码
git add .
git commit -m "修改了什么"
git push origin master
```

GitHub Actions 会自动：
1. 服务器上的 Runner 收到任务
2. 拉取最新代码
3. 安装依赖（如果有新增）
4. 构建前端
5. 同步到 `/opt/ams/`
6. 重启后端服务
7. reload nginx

全程无需再登录服务器。

## 五、常见问题

### Q: Runner 注册后 GitHub 上不显示 🟢 Idle？

A: 在服务器上检查 Runner 状态：
```bash
cd /opt/ams-runner
sudo ./svc.sh status actions-runner
```
如果没运行则启动：
```bash
sudo ./svc.sh start actions-runner
```

### Q: 后端启动失败？

A: 查看日志：
```bash
sudo journalctl -u ams-backend -n 50 --no-pager
```

### Q: 如何手动触发部署？

A: 在 GitHub 仓库页面 → **Actions** → 选择 `Deploy to Self-Hosted Runner` → **Run workflow** → 选择分支 → 点击 **Run workflow**
